### What to include

- Affected version (commit hash or release tag)
- Affected Android version(s) and device(s) tested
- A clear description of the vulnerability and its impact
- Proof-of-concept code or steps to reproduce
- Suggested mitigation, if you have one

### Response timeline

- **Acknowledgement**: within 72 hours of report
- **Initial assessment**: within 7 days
- **Coordinated disclosure**: typically 90 days from report,
  earlier if the issue is critical and a fix is ready

We will keep you informed throughout the process and credit you in the
release notes (unless you prefer to remain anonymous).

## Scope

The following are **in scope**:

- All code in this repository
- Default build configurations
- Security claims made in the README (encryption, key handling, persistence, wipe behavior)

The following are **out of scope**:

- Vulnerabilities in upstream dependencies (please report directly to upstream)
- Vulnerabilities requiring physical access to an already-unlocked device
- Issues that require malicious apps with elevated privileges already installed
- Theoretical attacks without working proof-of-concept
- DoS through obvious resource exhaustion (battery drain via excessive wipes etc.)
